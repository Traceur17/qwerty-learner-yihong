# 饼干词库音频后台排队（设计）

日期：2026-07-13  
范围：仅 `wang-c3/4/5/11-biscuit` 四本专属词库。

## 目标

在听写练习中，尽快让当前章可练，并在不抢占带宽的前提下，把四本专属词库的自定义 MP3 暖进磁盘 Cache。

## 行为

1. **本章（前台）**  
   缺文件时阻塞并显示「缓存音频 x/y」；已缓存则立刻可练。

2. **本词库其它章（后台）**  
   本章就绪后，按「后续章 → 更早章」顺序暖磁盘；已同 `av` 缓存则跳过。  
   底部保留小字：`后台缓存后续章节音频…`

3. **其它饼干词库（后台）**  
   本词库排完后，按 **C3 → C4 → C5 → C11** 排队（跳过当前词库）。  
   小字示例：`后台缓存 C5 词库音频…`  
   非饼干词库不触发跨库预加载。

4. **切章优先**  
   切换章节/词库时中止后台队列，立刻优先 ensure 新章；再重新排后台。

5. **约束**
   - 后台低并发（低于前台）
   - 不重复下载已在磁盘且同世代的文件
   - 仅 `av` 变更后才重拉

## 默认听写

`listenDictationConfig.isOpen` 默认改为 `true`（新用户 / 无本地配置时）。已有 localStorage 的老用户保留其开关状态。

## 非目标

- 进站立刻全量拉四本（不阻塞首页）
- 改练习记录 / IndexedDB 之外的业务数据
- 对非饼干词库做跨库预加载
