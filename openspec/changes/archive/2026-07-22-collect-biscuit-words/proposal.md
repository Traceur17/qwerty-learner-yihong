## Why

练习时经常在站外雅思听力/阅读材料里遇到生词表，需要截图或粘贴后快速入库，并自动配好音标、释义与发音，再回到原练习页继续。开源站目前只有预置静态词库，没有「页面上收集 → 补全 → 可练」的闭环；本变更面向私有单用户场景，用 Gemini（用户自备 key）做识图/补全，本地保存自定义词库「收集小饼干」。

## What Changes

- 新增固定自定义词库 **「收集小饼干」**，新收集的词默认写入该库，并出现在词库选择中可练习
- Header：**点饼干图标**打开收集浮层；**点标题文字**仍回首页（方案 A）
- 收集浮层：支持打字、粘贴/选择图片；回车触发识别；展示可勾选/可编辑的单词卡片（音标、释义、发音试听）；默认确认保存；可重新识别
- 保存成功：浮层缩成小饼干飞入「饼干盒」动画，徽章显示「收集小饼干」词数，然后关闭浮层回到原页面
- 词库详情支持对该库做批量添加 / 批量识别（同一套识别管线）
- 设置页：Gemini API key 的保存、清除、连通测试（BYOK，仅存浏览器本地）
- 补全：词典源自动填音标与中文释义并直接展示；词典 miss 时用 Gemini 补；释义允许手改
- 重复词：若词已存在于任意词库，提示所在词本，由用户决定是否仍写入「收集小饼干」
- 一次多词：卡片全部默认勾选，用户可取消部分后再保存

## Capabilities

### New Capabilities

- `biscuit-word-collection`: 收集浮层、Logo 点击拆分、识别结果卡片、保存动画与饼干盒词数徽章、批量添加入口
- `custom-collected-dictionary`: 「收集小饼干」本地持久化、与 Gallery/练习流集成、词数统计
- `ai-assisted-word-enrichment`: Gemini BYOK 设置、截图/文本识词、音标释义补全与发音试听管线

### Modified Capabilities

- （无）现有练习/连播等能力的需求不变；本变更只新增收集与自定义词库能力

## Impact

- **UI**: `Header` / `BiscuitIcon` 点击行为；新增收集弹窗与设置中的 Gemini 区块；Gallery 需展示「收集小饼干」
- **数据**: IndexedDB（Dexie）新增自定义词条存储；练习时需能加载该库词表（不依赖 `public/dicts` 静态 JSON）
- **外部依赖**: Google Gemini API（用户 key）；词典补全（如 Free Dictionary API / 本地 ECDICT 等，见 design）；发音复用现有有道 `dictvoice`
- **隐私**: API key 仅存用户本地；不写入仓库；私有单用户场景不部署服务端代理
